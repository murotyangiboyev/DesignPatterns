# Singleton Pattern

### 🎯 What problem it solves
The **CreationalPatterns.Singleton** pattern ensures that a class has only one instance and provides a global point of access to it.


* Prevents accidental creation of multiple instances.
* Controls resource usage efficiently (e.g., memory, connections).
* Can simplify coordination across different parts of an application.


### 🎨 Features

* **CreationalPatterns.Singleton instance:** Ensures only one object of the class exists in the JVM.
* **Global Access Point:** Provides a centralized way to access the instance.
* **Thread safety:** Can be designed to work correctly in multithreaded environments
* **Resource Managment:** Useful for managing shared resources like configurations, logging, or database connections.

### Use cases

* Consider using the CreationalPatterns.Singleton pattern when you need to ensure that only one instance of a class exists in your application
